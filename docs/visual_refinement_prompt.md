# 현 버전 시각 인지 보완 실행 프롬프트

## 목적

성남 10분 생활필수 접근권 대시보드의 데이터, 점수, 분류 로직은 유지하고, 심사자가 색상 의미와 화면 위계를 더 빠르게 이해하도록 시각 인지 구조를 정리한다.

## 실행 지시

1. 상태 색상과 카테고리 색상을 분리한다.
   - 우선 점검, 주의 관찰, 대체로 양호 같은 상태값은 지도 점수 레이어에서만 사용한다.
   - 의료, 약국, 교통, 돌봄 같은 서비스 카테고리는 중립 톤의 명도 차이로만 구분한다.
   - 정책 유형 카드의 강조색도 위험 신호처럼 보이지 않도록 중립 톤으로 낮춘다.

2. 지도 라벨은 지도 데이터를 가리지 않게 정리한다.
   - 불투명한 라벨 박스를 줄이고 반투명 배경과 텍스트 그림자를 사용한다.
   - 지도 색상, 서비스 점, 배경 지형이 라벨에 과도하게 가려지지 않도록 한다.

3. 산점도 기본 title 툴팁을 제거한다.
   - 브라우저 기본 툴팁 대신 hover/focus 시 우측 프로필 카드가 즉시 해당 생활권 정보로 바뀌게 한다.
   - 클릭은 기존 선택 기능으로 유지한다.

4. 정책 카드의 텍스트 벽을 줄인다.
   - 데이터 신호, 단기 점검, 중기 검토 라벨은 배지형 키워드로 강조한다.
   - 본문 안 핵심어는 굵게 표시해 빠르게 스캔할 수 있게 한다.

## 준수 조건

- CSV, JSON, 점수 산식, 평균값, 우선 확인 생활권, 정책 보완 유형 목록은 변경하지 않는다.
- 변경 범위는 `AccessMap`, `NeedAccessExplorer`, `DashboardV2`, 전역 CSS에 한정한다.
- 반영 후 `npm run lint`, `npm run build`, Playwright 캡처를 수행한다.
