# v1 / v2 비교

## 접근 경로

- `/v1`: 기존 버전. 기능 검증용 화면으로 보존.
- `/v2`: 디자인 개선 버전. 공모전 제출용 화면.
- `/`: v2와 동일한 제출용 기본 화면.

## v1

- 정보 구조와 기능 흐름을 검증하기에 적합하다.
- 지도, 서비스별 비교, 우선순위 산점도, 정책 보완 카드가 모두 한 흐름 안에 들어 있다.
- 다만 카드 내부 여백과 줄간격이 촘촘하고, 일부 라벨과 배지 반복이 생성형 AI 템플릿처럼 보일 수 있다.

## v2

- 기능과 데이터 구조는 유지한다.
- 타이포그래피, 여백, 카드 위계, 지도 명도를 조정했다.
- 첫 화면은 메시지와 대표 지표, 두 번째 화면은 서비스별 차이, 세 번째 화면은 우선순위, 네 번째 화면은 정책 보완 방향으로 역할을 분리했다.
- 방법론은 기존 모달 진입점을 유지하되 버튼과 모달 톤을 차분하게 정리했다.

## 항목별 비교

| 항목 | v1 | v2 |
| --- | --- | --- |
| 줄간격 | 본문과 카드 설명이 비교적 촘촘해 긴 문장이 한 덩어리로 보인다. | 본문과 설명 line-height를 넓히고 문단 폭을 제한해 발표 화면에서 읽기 쉽다. |
| 여백 | 섹션, 카드, 리스트 사이 간격이 좁아 정보가 몰려 보인다. | 섹션 상단, 지도-패널 간격, 카드 내부 padding을 키워 화면 호흡을 확보했다. |
| 카드 가독성 | KPI, 분석 범위, 우선 점검 리스트가 같은 밀도로 이어진다. | KPI, 분석 범위, 우선 점검 리스트를 다른 카드 위계와 행 높이로 분리했다. |
| 지도 가시성 | 지도 정보는 충분하지만 범례와 보조 설명이 다소 경쟁한다. | 지도 배경은 낮추고 주제 레이어를 선명하게 조정했으며, 서비스 화면은 공통 범례를 사용한다. |
| 정책 카드 가독성 | 항목이 표처럼 빽빽하게 나열되어 정책 브리프 느낌이 약하다. | 2x2 정책 브리프 카드로 바꾸고 항목 간격과 문장 길이를 줄였다. |
| 발표자료 활용성 | 기능 검증에는 적합하지만 16:9 캡처에서 일부 화면이 과밀하다. | 1920x1080 발표 모드 캡처를 별도 생성했고, 핵심 메시지와 대표 시각화가 먼저 보인다. |
| AI 템플릿 느낌 감소 | 반복 배지, 둥근 pill, 직접적인 결론 라벨이 반복된다. | 라벨을 “핵심 해석”, “정책적 의미”, “읽는 법”으로 바꾸고 그림자와 장식을 줄였다. |

## 검수 포인트

- 제목은 2줄 이내로 자연스럽게 보이도록 조정했다.
- 본문 line-height는 v2 스코프에서 넓게 적용했다.
- KPI 숫자와 설명 사이 간격을 분리했다.
- 지도와 우측 패널의 간격을 넓혔다.
- 정책 카드의 항목 간격을 넓히고 구분선 의존도를 줄였다.
- v1은 Git 태그 `v1-current`와 `/v1` 라우트로 함께 보존했다.
