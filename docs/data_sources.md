# 데이터 출처 및 매핑 구조

## 1. 입력 파일

`data/accessibility_metrics.csv`

행정동 단위 분석 지표다. 주요 컬럼은 `area_id`, `area_name`, `district`, `lat`, `lng`, `population`, `elderly_ratio`, `single_ratio`, `care_demand`, `medical_score`, `pharmacy_score`, `transit_score`, `care_score`, `overall_score`, `vulnerable_index`, `selection_note`다.

`data/service_points.csv`

서비스 위치 데이터다. 주요 컬럼은 `service_type`, `lat`, `lng`, `name`, `district`, `area_name`이다. `service_type`은 `medical`, `pharmacy`, `bus`, `care` 중 하나를 사용한다.

## 3. 현재 데이터 상태

현재 CSV는 공모전 발표용 대표 생활권 10개 동 데이터다. 실제 API 또는 성남시 실시간 데이터와 자동 연동된 구조는 아니며, 동일 컬럼의 최신 CSV로 교체하면 전체 행정동 전수 분석으로 확장할 수 있다.

## 4. 교체 원칙

- 동별 좌표는 행정동 중심점 또는 인구가중 중심점을 사용한다.
- 서비스 위치는 위도·경도 좌표를 포함해야 한다.
- 모든 서비스별 점수는 0~100 범위로 표준화한다.
- 기준시점이 다른 데이터는 가장 최신 공공데이터 기준으로 재산정하고, 기준시점을 문서에 남긴다.
