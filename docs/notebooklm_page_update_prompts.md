# NotebookLM 페이지별 수정 프롬프트

전제: NotebookLM 소스에 `public/reports/seongnam_10min_access_v2_actual_data_capture.pdf`를 함께 넣는다. 아래 프롬프트는 공통 수정사항을 반복하지 않고, 각 발표 페이지에서 바꿔야 할 내용만 지시한다.

## 1쪽. 문제 배경 / 표지

```text
첨부한 최신 시각화 PDF 1쪽을 기준으로 이 페이지의 메시지를 수정해줘.

반영할 내용:
- 제목은 “성남 10분 생활필수 접근권 격차 지도”로 유지한다.
- 핵심 문장은 “생활서비스가 있는 것과 10분 안에 닿는 것은 다릅니다.”로 정리한다.
- 문제 배경은 “서비스 개수”보다 “체감 도달성”이 정책 우선순위를 가른다는 방향으로 바꾼다.
- 지도에서 대표 생활권 평균 접근성이 70점으로 갱신된 점을 반영한다.
- 너무 단정적으로 “접근권 부족”이라고 쓰지 말고, “대표 생활권 간 접근성 차이를 비교했다”는 표현을 사용한다.

출력은 이 페이지에 들어갈 제목, 부제, 핵심 메시지, 2문장 발표 멘트로 나눠 작성해줘.
```

## 2쪽. 사용 데이터

```text
최신 시각화 PDF와 데이터 출처 문서를 기준으로 이 페이지의 데이터 설명을 수정해줘.

반영할 내용:
- 의료기관, 약국, 노인종합복지관, 인구·세대, 1인세대는 성남시·공공데이터포털 공개 파일을 내려받아 반영했다고 쓴다.
- 교통은 경기교통정보센터의 성남시 2025년 버스정류소 집계와 OpenStreetMap 실제 버스정류장 좌표를 결합했다고 수정한다.
- 이전 표현 중 “버스공영차고지 기반 대리 지표”는 삭제한다.
- 공모전 제공 민간 통신·카드 데이터는 제공 경로와 활용 가능성은 확인했지만, 현재 앱 CSV에는 직접 반영하지 않았다고 투명하게 쓴다.
- 데이터가 모두 실제 공개 원천에서 왔다는 점과, 보행 네트워크 이동시간은 아직 추가 확장 단계라는 점을 함께 설명한다.

출력은 슬라이드 본문용 4개 bullet과 발표자 노트 3문장으로 작성해줘.
```

## 3쪽. 지표 산식

```text
최신 시각화 PDF와 산식 문서를 기준으로 이 페이지의 산식 설명을 수정해줘.

반영할 내용:
- 종합 접근성 가중치는 의료 30%, 약국 20%, 교통 25%, 돌봄 25%로 유지한다.
- 의료·약국 점수는 대표 동에 매핑된 실제 기관 수를 0~100으로 표준화한 값이라고 설명한다.
- 교통 점수는 실제 버스정류장 좌표와 대표 동 중심점의 직선거리 기반이라고 설명한다.
- 돌봄 점수는 가장 가까운 노인종합복지관까지의 거리 기반이라고 설명한다.
- 지원수요는 고령층 비율, 1인세대 비율, 고령 1인세대 비율을 결합한 값이라고 수정한다.
- “실제 보행 10분 이동시간”으로 단정하지 말고 “위치 기반 접근성 지표”라고 표현한다.

출력은 슬라이드용 짧은 산식 설명, 시각화 주석 문구, 심사 질의 대응 멘트로 나눠 작성해줘.
```

## 4쪽. 분석 결과 / 한눈에 보기

```text
최신 시각화 PDF 1쪽을 기준으로 이 페이지의 결과 숫자와 해석을 갱신해줘.

반영할 내용:
- 대표 생활권 평균 접근성은 70점으로 수정한다.
- 10개 대표 생활권의 지도 해석은 “대체로 양호하지만 생활권별 보완 축이 다르다”로 정리한다.
- 우선 점검 순서는 지원수요까지 함께 본 결과로 태평1동, 수진2동, 은행2동, 상대원1동, 구미동이 먼저 보인다고 반영한다.
- 태평1동은 종합 69.3점, 지원수요 96점으로 우선 검토 대상으로 설명한다.
- 지도는 “서비스가 많은 곳”이 아니라 “실제 닿기 쉬운 곳과 먼저 보완할 곳”을 같이 보여준다고 설명한다.

출력은 슬라이드 제목 1개, 핵심 문장 1개, 숫자 콜아웃 3개, 발표자 노트 4문장으로 작성해줘.
```

## 5쪽. 서비스별 부족 / 무엇이 부족한가

```text
최신 시각화 PDF 2쪽을 기준으로 이 페이지의 서비스별 부족 지역을 갱신해줘.

반영할 내용:
- 의료 접근성 TOP 3 부족 생활권은 복정동 48점, 은행2동 54점, 상대원1동 59점이다.
- 약국 접근성 TOP 3 부족 생활권은 복정동 48점, 은행2동 58점, 상대원1동 62점이다.
- 교통 접근성 TOP 3 부족 생활권은 금광1동 48점, 정자동 58점, 복정동 66점이다.
- 돌봄 접근성 TOP 3 부족 생활권은 야탑3동 48점, 구미동 57점, 태평1동 65점이다.
- “같은 생활권이라도 부족한 서비스가 다르다”는 메시지를 강조한다.
- 교통은 이제 실제 정류장 좌표 기반이므로 공영차고지 관련 문구는 쓰지 않는다.

출력은 서비스별 1줄 해석 4개와 발표자가 말할 비교 해석 3문장으로 작성해줘.
```

## 6쪽. 우선순위 / 어디가 우선인가

```text
최신 시각화 PDF 3쪽을 기준으로 이 페이지의 우선순위 해석을 수정해줘.

반영할 내용:
- 산점도 해석은 “접근이 어렵고 지원수요가 높은 생활권부터 먼저 점검한다”로 유지한다.
- 선택 생활권은 태평1동으로 설명하고, 종합 접근성 69점, 접근 어려움 31점, 지원수요 96점을 반영한다.
- 태평1동의 우선 확인 축은 돌봄이며, 고령층 27%, 1인가구 38%로 설명한다.
- 태평1동은 서비스 접근성이 최저인 곳이라기보다 지원수요가 매우 높아 정책 우선순위가 올라간 사례라고 해석한다.
- 복정동·금광1동처럼 서비스 점수는 낮지만 지원수요 구조가 다른 곳과 구분해 설명한다.

출력은 슬라이드 제목, 산점도 읽는 법, 태평1동 프로필 문구, 발표자 노트 4문장으로 작성해줘.
```

## 7쪽. 정책 유형 / 어떻게 보완할까

```text
최신 시각화 PDF 4쪽을 기준으로 이 페이지의 정책 유형과 대상 생활권을 수정해줘.

반영할 내용:
- 의료 접근 보완형 대상은 수진2동, 복정동, 은행2동, 상대원1동이다.
- 이동 접근 보완형 대상은 금광1동, 정자동이다.
- 돌봄 접근 보완형 대상은 태평1동, 서현1동, 야탑3동, 구미동이다.
- 복합 보완 필요형은 현재 대표 생활권에서 단독 대상이 없으면 빈 유형으로 보이지 않게, “복합 보완 필요 생활권이 없으면 서비스별 우선 점검 카드로 나눠 검토”라고 해석한다.
- 이동 접근 보완형의 데이터 신호는 실제 정류장 좌표 기준 교통 점수가 낮고 생활서비스 이동 의존도가 높다는 내용으로 수정한다.
- 정책 제안은 “순위”가 아니라 “부서별 보완 방향”으로 이어진다는 메시지를 강조한다.

출력은 유형별 카드 문구 4개와 슬라이드 하단 결론 문장 1개로 작성해줘.
```

## 8쪽. 한계와 확장

```text
최신 시각화 PDF와 데이터 출처 문서를 기준으로 이 페이지의 한계와 확장 계획을 수정해줘.

반영할 내용:
- 현재 데이터는 실제 공개 원천을 내려받아 앱 CSV로 재산출한 최신본이라고 쓴다.
- 교통은 실제 정류장 좌표와 대표 동 중심점의 직선거리 기반이며, 보행 네트워크 이동시간은 아직 반영하지 않았다고 설명한다.
- 공모전 제공 민간 통신·카드 데이터는 현재 앱 CSV에 직접 반영하지 않았고, 확보 시 체류·소비·방문 강도 보정 지표로 추가한다고 쓴다.
- 향후 보행 네트워크, 경사, 횡단 대기, 야간·휴일 운영 여부를 붙이면 실제 10분 도달성 지표로 확장 가능하다고 정리한다.
- “데이터가 가짜”라는 인상을 주지 않도록, 실제 반영 데이터와 추가 확장 데이터를 명확히 구분한다.

출력은 슬라이드 본문 5개 bullet과 심사 질의 답변용 3문장으로 작성해줘.
```
