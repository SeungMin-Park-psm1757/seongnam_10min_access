# design v2 notes

## 디자인 방향

v2는 공공정책 보고서와 인터랙티브 지도 작품 사이의 톤을 목표로 한다. 과한 장식보다 넓은 여백, 읽기 쉬운 한국어, 절제된 색상, 지도 중심의 시각적 앵커를 우선했다.

## 변경 이유

- v1은 정보 구조가 좋지만 카드 간격과 내부 여백이 좁아 발표 화면에서 과밀하게 보인다.
- 서비스별 small multiple과 정책 카드가 모두 비슷한 카드 문법을 사용해 템플릿처럼 보일 가능성이 있다.
- 지도, KPI, 우선 점검 리스트의 위계가 조금 더 분명해야 첫 화면의 메시지가 빠르게 읽힌다.

## 변경 전후 포인트

- `/v1`은 기능 검증용 기존 화면으로 보존했다.
- `/v2`와 `/`는 제출용 디자인 개선본으로 연결했다.
- Hero 제목과 본문 사이 간격을 넓히고 “결론:” 라벨을 “핵심 해석”, “정책적 의미”, “읽는 법”으로 바꿨다.
- 본문과 카드 설명의 줄간격을 넓히고 문단 폭을 제한했다.
- 우측 패널은 KPI, 산정 방식, 분석 범위, 우선 점검 리스트, 데이터 출처가 여백으로 구분되도록 조정했다.
- 지도 배경은 더 연하게, 접근성 원과 서비스 점은 더 선명하게 조정했다.
- 정책 카드는 표처럼 보이지 않도록 항목 간 간격을 키우고 구분선을 줄였다.
- 카드 둥근 모서리는 8px 중심으로 낮추고 그림자 사용을 줄여 보고서형 화면에 가깝게 정리했다.

## 2차 정교화

- 화면 1은 대표 KPI, 분석 범위, 우선 점검 리스트 중심으로 첫 시선을 정리했다.
- 화면 2는 서비스별 부족 TOP 5를 TOP 3로 줄이고, 공통 범례와 짧은 정책 힌트를 적용했다.
- 화면 3은 축 제목을 쉬운 문장으로 바꾸고 계산식은 화면에서 덜어냈다.
- 화면 3의 프로필 카드는 동 이름, 한 줄 해석, 핵심 지표 카드, 지원수요 미터바로 재구성했다.
- 화면 4는 2x2 정책 브리프 카드로 정리하고 “왜 필요한가” 설명을 줄여 데이터 신호와 실행 과제를 먼저 읽게 했다.
- 방법론 모달은 사용 데이터, 지표 산식, 점수 구간, 대표 동 선정, 한계와 확장, 데이터 갱신 가능성으로 재구성했다.
- 발표 모드에서는 보조 카드와 방법론 버튼을 숨기고 대표 지도, KPI, 서비스 카드, 정책 브리프가 먼저 보이도록 조정했다.

## 색상

- 배경: 깨끗한 오프화이트 `#faf8f2`
- 카드: 따뜻한 백색 `#fffefa`
- 강조: 네이비, 청록, 세이지, 적갈색, 블루그레이
- 위험/우선 점검: 강한 빨강 대신 낮은 brick 계열

## 남은 디자인 이슈

- 실제 성남시 전수 행정동 데이터가 들어오면 정책 카드별 지역 수가 늘어날 수 있어 목록 접힘 또는 상세 보기 패턴이 추가로 필요하다.
- 지도 레이블은 현재 대표 10개 동 기준에 맞춰 조정되어 있어 전수 데이터에서는 밀도 기반 표시 정책이 필요하다.
- 발표용 스크린샷에서는 문제 없지만, 모바일 대응은 현재 공모전 발표 해상도 중심으로 제한되어 있다.
