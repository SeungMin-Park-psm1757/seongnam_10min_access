# 워딩 보완 정리

## 적용 방향

기능과 데이터 구조는 유지하고, 화면에 보이는 한국어 문구만 공모전 발표용 톤으로 정리했다.

핵심 방향은 다음과 같다.

- 기술 설명보다 정책 질문과 판단 기준이 먼저 보이도록 조정
- 취약·공백 같은 낙인성 표현은 필요한 곳에만 사용
- 점수는 숫자보다 해석 문구와 함께 제시
- 정책 제안은 단정형보다 우선 점검, 검토, 보완 중심으로 표현
- 화면 제목은 발표용 섹션명으로 순화

## 주요 문구 비교

| 구분 | 기존 문구 | 수정 문구 |

| 네비게이션 | 서비스 | 무엇이 부족한가 |
| 네비게이션 | 우선 | 어디가 우선인가 |

## 화면별 한 줄 결론

1. 한눈에 보기: 생활서비스의 수보다 실제로 닿기 쉬운지가 정책 우선순위를 가릅니다.
2. 무엇이 부족한가: 같은 생활권이라도 먼저 보완할 서비스는 다를 수 있습니다.
3. 어디가 우선인가: 접근성이 낮고 지원수요가 높은 곳이 우선 점검 대상입니다.
4. 어떻게 보완할까: 정책은 순위가 아니라 유형별 보완 방향으로 이어져야 합니다.

## 정책 카드 최종 구조

각 정책 카드는 다음 구조로 통일했다.

- 왜 보완이 필요한가
- 데이터가 보여주는 신호
- 먼저 할 수 있는 일
- 중기 검토 과제
- 협업 부서

## 방법론 문구 최종안

본 지표는 의료·약국·교통·돌봄 접근성을 0~100점으로 표준화해 비교한다. 현재 값은 위치 기반 근사 지표이며, 실제 정책 적용 전에는 현장 확인과 추가 데이터 검증이 필요하다.

점수 구간은 다음과 같이 해석한다.

- 70점 이상: 대체로 양호
- 58~69점: 주의 관찰
- 57점 이하: 우선 점검 필요

## 데이터 연동 표현 주의

현재 화면은 성남시 실시간 API 또는 최신 공공데이터와 직접 연동된 결과가 아니다. 대표 표본 CSV를 기반으로 한 시각화이므로 제출 문구에서는 “실제 데이터 연동”보다 “공공데이터로 확장 가능한 분석 구조”, “대표 표본 기반 정책지도”로 표현하는 것이 안전하다.
